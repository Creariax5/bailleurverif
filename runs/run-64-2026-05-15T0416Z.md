## FINDINGS

**(1) Hestia** : `User-Agent: *` `Allow: /` + 14 `Disallow` paths transactionnels (espaces bailleur/locataire/mandataire, /api/, /auth/, /bail/creer, /bail/signing/, /etat-des-lieux/creer, /etat-des-lieux/signing/, /quittance/creer, /assurances/pno, /assurances/mrh, /avenant/signing/, /partenariat-iad/) + Sitemap. **Aucune whitelist IA explicite** (pas de `GPTBot`, `ClaudeBot`, etc. listés positivement). Comportement par défaut : crawlers IA accèdent à toute la partie publique mais sans signal sémantique "vous êtes les bienvenus".

**(2) Qlower** : `301 Moved Permanently` vers une URL qui ne sert QUE `Sitemap: https://www.qlower.com/sitemap.xml` (pas de directives Allow/Disallow/User-agent). **Robots.txt non-conforme RFC** — manque le corps directif standard. Pour les crawlers obéissants, cela revient probablement à "tout autorisé par défaut", mais c'est un drapeau de qualité technique faible.

**(3) Rentila (www)** : `User-Agent: *` `Allow: /` + `Disallow: /payments/` + **block dotbot explicite** (`User-agent: dotbot` `Disallow: /`). Pas de whitelist IA explicite. Pattern défensif observable : block crawler agressif spécifique. Notre robots.txt ne bloque pas dotbot (à considérer si on subit du scraping).

**(4) Maslow** : `Access Denied` via Akamai edgesuite.net même avec UA Firefox ET UA `GPTBot/1.0` (testés sur `www.maslow.immo` ET `maslow.immo`). **Robots.txt inaccessible publiquement**. RFC précise que 4xx/5xx sur robots.txt est interprété par les bots respectueux comme "accès libre", mais Access Denied + Akamai derrière = signal mixte. Hypothèse : Akamai filtrage agressif sur path `/robots.txt` peut sembler "no robots.txt" → certains bots IA pourraient skip totalement leur site. **Anti-pattern Maslow** : leur Patch #6 hypothétique (bloc résumé IA) peut être miné si les bots IA ne parviennent pas à crawler à cause de robots.txt 403.
