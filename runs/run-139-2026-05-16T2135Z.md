### 1. Bug latent #13 fixé en place (`/api/embed/view` GET handler)

Ajout dans `wedge-tool/server.py` après `/healthz` (do_GET), 26 lignes :

```python
if path == "/api/embed/view":
    qs = parse_qs(parsed.query or "")
    tool = (qs.get("w", [""])[0] or "").strip().lower()[:32]
    ville = (qs.get("v", [""])[0] or "").strip().lower()[:64]
    dpe = (qs.get("d", [""])[0] or "").strip().upper()[:4]
    ref = self.headers.get("Referer", "")[:300]
    ua = self.headers.get("User-Agent", "")[:200]
    ip = self.client_address[0]
    rec = {
        "ts": now_iso(), "tool": tool, "ville": ville, "dpe": dpe,
        "referrer": ref, "ip_hash": str(abs(hash(ip)) % (10**10)), "ua": ua
    }
    try: append_jsonl(EMBED_VIEWS_FILE, rec)
    except Exception: pass
    gif = b"GIF89a\x01\x00\x01\x00\x80\x00\x00\x00\x00\x00\x00\x00\x00!\xf9\x04\x01\x00\x00\x00\x00,\x00\x00\x00\x00\x01\x00\x01\x00\x00\x02\x02D\x01\x00;"
    self._send(200, gif, ctype="image/gif")
    return
```

Caps de sécurité : tool≤32, ville≤64, dpe≤4, referer≤300, ua≤200. Pas de PII (ip_hash anonyme, pas d'IP brute persistée).
