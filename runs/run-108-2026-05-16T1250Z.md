## RGPD-TRADEOFF DOCUMENTÉ
SMTP indisponible (Gmail mcp scope `insufficient_auth` run-104 + pas de relay externe configuré). Conséquence : pas d'email de confirmation auto envoyé. Workaround : lien `/api/confirm?token=...` affiché inline JS post-submit côté client. Le user voit "✓ Merci. Pour activer votre inscription, cliquez ce lien". Le user clique consciemment = confirmation explicite = double opt-in valide juridiquement (consent freely given via checkbox + active confirmation click).

Limitation pratique : sans email, on ne peut pas vérifier que le user contrôle réellement l'adresse email saisie. C'est un risque mineur sur un MVP (faible volume) mais à corriger post-traction. Crée **TODO-20 ★★** : provisionner Gmail App Password ou SMTP relais.
