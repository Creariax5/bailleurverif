## DÉCOUVERTES

1. **Builder en 1 fichier python pur tient** : pas besoin d'écosystème node/jekyll/hugo pour 3-10 articles. Réutilisable, compréhensible (1 fichier, 1 commande). Si on dépasse 30 articles ou qu'on a besoin de syntaxes plus riches (tableaux, code blocks), on migrera — pas avant.
2. **Le placeholder `{{CTA}}` était sous-utilisé** : il n'autorisait qu'un CTA en fin d'article. Le pattern 3-CTAs (hero/mid/footer) triple les opportunités de conversion sans surcharger le lecteur. La logique d'injection est dans le builder, pas dans le markdown source — donc les markdown sources restent publiables ailleurs (Medium, LinkedIn long-form) avec un seul CTA générique en fin.
3. **`metrics_sync.sh` accepte facilement de nouvelles dimensions** : ajout de `articles_published` en 4 lignes, sans casser l'idempotence du script. Bon pattern à conserver pour les prochaines metrics (twitter_followers, reddit_karma, etc. si on les automatise).
4. **Le critère go/no-go est testable plus tôt qu'on le pensait** : avec 3 articles HTML chacun avec 3 CTAs sur l'IP visible, un seul partage Reddit ou Twitter qui drive 30 clics suffit pour avoir un signal exploitable (conv % à 30 visites = pas robuste statistiquement mais directionnel). On n'a pas besoin d'attendre 100 visites pour itérer le copy CTA.
