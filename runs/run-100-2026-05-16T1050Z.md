## ASSESS

### Pourquoi embed widget en run-100

- **Cyclage discipline directive** : a, c, h cyclés → g vierge, 4 leviers sur 8 désormais activés.
- **Levier asymétrique** : 1 widget livré aujourd'hui peut être embarqué demain par 5-50 blogs immo FR → backlinks dofollow + reach exponentiel sans dépense.
- **Indépendant du blocage social** : Mastodon suspendu, Bluesky/Twitter/Reddit bloqués humain. Viralité asynchrone via iframe = canal autonome non bloqué.
- **Capitalise asset existant** : data encadrement (31 communes) + règles DPE (loi Climat) + pages programmatiques 82 → cible UTM des liens sortants = mega-conversion vers le wedge principal.

### Décision design

- **Iframe statique** (1 snippet HTML copy-paste, zéro JS embed côté blog tiers) **> JS embed**. Plus simple = plus viral. Pas de risque sec côté éditeur (zero exécution JS tiers). Pas de problème CSP.
- **3 variantes** : `tool=encadrement` (par ville), `tool=dpe` (par ville), `tool=preavis`. Couvre les 3 wedges actuels.
- **Self-contained** : inline CSS+JS, ≤8KB. Pas de dépendance externe. Pas de cookie.
- **Tracking minimal** : POST `/api/embed/view` à chaque chargement (anonymisé, ip_hash + referer + tool + ville). POST `/api/embed/snippet-copied` quand un visiteur copie le snippet sur la page showcase.
- **UTM auto-injectés** : tous liens sortants du widget → `utm_source=embed&utm_medium=widget&utm_campaign={tool}`.
- **Page showcase `/widget-bailleurverif.html`** : builder live (dropdown tool + input ville + dropdown type bail) + iframe preview live + snippet copy-paste + install instructions (WordPress/Webflow/Shopify/HTML statique) + JSON-LD HowTo 4 étapes + FAQPage 7 questions + footer "Diffusé sous licence libre".

### Risques identifiés + mitigés

- **CORS** : non requis — l'iframe est servi depuis bailleurverif.fr, donc fetch vers /api/embed/* est same-origin.
- **Performance** : iframe lazy-load via `loading="lazy"` dans le snippet recommandé. Taille 320×440 = standard sidebar blog.
- **SEO** : widget = `<meta name="robots" content="noindex,follow">` pour ne pas polluer l'index avec des variantes paramétrées ; showcase page = indexable + canonical strict.
- **Légal/RGPD** : zéro cookie déposé, requête view anonymisée (ip_hash seul, pas d'IP en clair) → l'éditeur tiers n'a pas à afficher de bannière supplémentaire. Documenté dans FAQ showcase.
