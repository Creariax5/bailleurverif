## Découvertes

1. **Discipline méta : "agir maintenant" > "attendre l'horloge"**. Run-33→37 ont attendu T+3h mécaniquement. Run-38 a critiqué ce pattern avec l'idée "API publique gratuite". Run-40 va plus loin : *même quand une session BB est nécessaire, attendre une heure parce qu'un protocole disait T+3h, alors que la baseline d'engagement est stable à 0 et que le profil fantôme bloque activement, c'est de la procrastination déguisée*. Pattern à généraliser : dès qu'un milestone bloque une métrique observable (ici : profil fantôme bloque conversion vue→fav), exécuter immédiatement. T+3h n'a de sens que pour des mesures qui *évoluent* avec le temps.

2. **Le bouton "Enregistrer les modifications" est universel sur Mastodon piaille.fr** (mais pas standardisé : POST-001 utilisait "Piailler"). Pattern button-dump heuristique multi-keyword + JS click confirmé robuste 2e fois. À encoder définitivement dans `agent-browser/fill_form.py` quand 3+ plateformes (premier candidat = `bluesky_post.py` au déblocage TODO-14).

3. **Toggles indexable/discoverable introuvables** : ni sur `/settings/profile` ni sur `/settings/preferences/other`. Probable cause : Mastodon piaille.fr (instance custom branding "Piaille") cache ces options dans un sub-panel collapsible ou dans `/settings/profile/migration` ou `/settings/profile/edit` (différent de `/settings/profile` qui est l'éditeur de bio). À investiguer en récupérant le HTML complet de `/settings/profile` (DOM dump des inputs visibles vs cachés). Non urgent — selon diagnostic run-38, indexable est secondaire vs profil fantôme (search Mastodon est opt-in mais hashtag timelines fonctionnent indépendamment).

4. **HTML enrichi côté Mastodon** : la note JSON envoyée fait 252 chars, l'API publique retourne 259 (HTML `<p>...</p>` wrap). Les liens dans `fields[]` sont auto-enrichis avec `<a href>` + tracking class. Comportement standard Mastodon, à savoir pour parser les bios à venir.

5. **Le compte est désormais "professionnellement présentable"** : un visiteur qui clique sur le profil voit (a) avatar + header brandés, (b) bio descriptive sans promo, (c) 3 fields informatifs dont le lien vers l'outil et la source officielle. Friction conversion attendue : nettement réduite. Le test sera POST-002 (encadrement loyer) sur un compte propre vs POST-001 publié sur compte fantôme.
